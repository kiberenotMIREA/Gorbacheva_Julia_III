# S03 – eda_cli: мини-EDA для CSV с HTTP-сервисом (HW04)

Небольшое CLI-приложение для базового анализа CSV-файлов с HTTP-сервисом оценки качества данных. Проект развивается от HW03 (CLI) к HW04 (HTTP API).

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## HTTP-сервис (HW04)

### Запуск сервера

```bash
uv run uvicorn eda_cli.api:app --reload --port 8000
```
Сервис будет доступен по адресу: http://localhost:8000

### Новый эндпоинт из HW04 (обязательный)

#### `POST /quality-flags-from-csv`
**Полный набор флагов качества из CSV-файла, включая новые эвристики из HW03.**

Этот эндпоинт является обязательным требованием HW04 и демонстрирует интеграцию новых эвристик качества данных из HW03 в HTTP-сервис. В отличие от базового эндпоинта `/quality-from-csv`, который возвращает только булевы флаги, данный эндпоинт предоставляет полную детализированную информацию о качестве данных.

**Принимаемые параметры:**
- `file` (обязательный, `multipart/form-data`): CSV файл для анализа
  - Поддерживаемые MIME-типы: `text/csv`, `application/vnd.ms-excel`, `application/octet-stream`, `text/plain`
  - Поддерживаемые расширения: `.csv`

**Возвращаемые данные:**
- `flags` (dict): Полный словарь всех флагов качества, включая:
  - Базовые флаги: `too_few_rows`, `too_many_columns`, `max_missing_share`, `too_many_missing`
  - Новые флаги из HW03:
    - `has_constant_columns` - наличие колонок с одинаковыми значениями
    - `constant_columns` - список константных колонок
    - `has_high_cardinality_categoricals` - категориальные признаки с высокой кардинальностью (>100 уникальных значений)
    - `high_cardinality_columns` - информация о колонках с высокой кардинальностью
    - `has_suspicious_id_duplicates` - проблемы с уникальностью ID-колонок
    - `suspicious_id_columns` - информация о колонках с подозрительными ID
  - `quality_score` - интегральная оценка качества (0.0-1.0)
- `quality_score` (float): Итоговая оценка качества данных
- `dataset_shape` (dict): Размеры датасета (`n_rows`, `n_cols`)
- `processing_time_ms` (float): Время обработки запроса в миллисекундах
- `filename` (str, optional): Имя загруженного файла

**Особенности и отличие от `/quality-from-csv`:**
1. Полные данные: Возвращает не только булевы флаги, но и детализированную информацию
2. Новые эвристики HW03: Включает все три новые эвристики, добавленные в HW03:
   - Проверка константных колонок
   - Обнаружение высокой кардинальности категориальных признаков
   - Проверка уникальности ID-колонок
3. Влияние на оценку: Новые эвристики учитываются при расчете `quality_score`:
   - Константные колонки: -0.15 к score
   - Высокая кардинальность: до -0.20 к score
   - Дубликаты ID: -0.25 к score

**Коды ответов:**
- `200 OK` - успешная обработка файла
- `400 Bad Request` - ошибка чтения CSV, пустой файл или неподдерживаемый формат

## Доступные команды CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```
Выводит базовую информацию о данных в консоль:

- количество строк и столбцов
- типы данных
- статистику по пропускам

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
```
Создаёт подробный отчёт с графиками и таблицами в указанном каталоге.

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.
## Новые параметры команды `report`
### Основные параметры:

- `out-dir` - каталог для отчёта (по умолчанию reports);
- `sep` - разделитель в CSV (по умолчанию ,);
- `encoding` - кодировка файла (по умолчанию utf-8).

### Новые параметры:

- `max-hist-columns` - максимум гистограмм для числовых колонок;
- `top-k-categories` - топ-N значений для категориальных признаков;
- `title` - заголовок отчёта в Markdown;
- `min-missing-share` - порог пропусков для проблемных колонок (0.0-1.0).

### Как параметры влияют на отчёт:

- `max-hist-columns` - ограничивает количество создаваемых гистограмм. Если в данных больше числовых колонок, создаются гистограммы только для первых N колонок;
- `top-k-categories` - определяет, сколько топ-значений показывать для каждой категориальной колонки. Например, если установить 10, будут показаны 10 самых частых значений;
- `title` - задаёт заголовок Markdown отчёта. Отображается как # Заголовок в начале файла;
- `min-missing-share` - порог для выделения проблемных колонок. Колонки с долей пропусков выше этого значения попадают в специальный раздел "Проблемные колонки".

## Примеры вызова `eda-cli report` с новыми опциями

### Пример 1: Базовая настройка
```bash
uv run eda-cli report data/example.csv \
  --title="Анализ клиентов" \
  --max-hist-columns=8 \
  --out-dir="clients_report"
```
### Пример 2: Детальный анализ категорий
```bash
uv run eda-cli report data/example.csv \
  --top-k-categories=10 \
  --min-missing-share=0.1 \
  --title="Детальный анализ"
```

### Пример 3: Быстрый анализ большого датасета
```bash
uv run eda-cli report large_dataset.csv \
  --max-hist-columns=3 \
  --out-dir="quick_analysis"
```
